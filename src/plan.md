# artgen 第一阶段计划

## 项目目标
先完成一个最小可运行的终端艺术字生成器：

输入一段文本，程序在终端输出大号 ASCII 艺术字，并支持单色显示。

这一阶段重点是先打通完整流程，不追求复杂字体、渐变或动画效果。

## 第一阶段范围
本阶段只实现以下能力：

- 命令行输入文本
- 使用 `figlet-rs` 生成大号 ASCII 艺术字
- 支持单色输出
- 在终端正确打印结果

本阶段不实现：

- 自定义 5x5 / 5x7 字模
- 渐变色
- 闪烁、跑马灯、流光等动画
- 复杂颜色解析

## 技术路线
第一阶段采用以下方案：

- `clap`：解析命令行参数
- `figlet-rs`：把普通文本转换成 ASCII 艺术字
- `crossterm`：控制终端颜色输出

这样可以先快速得到可运行结果，后续再扩展渐变和动画。

## 模块规划
项目目录保持如下结构：

```text
artgen/
├── Cargo.toml
└── src/
    ├── main.rs
    ├── cli.rs
    ├── render.rs
    └── color.rs
```

各模块职责：

- `main.rs`
  - 程序入口
  - 串联参数解析、艺术字生成、颜色输出

- `cli.rs`
  - 定义 CLI 参数
  - 解析用户输入的文本和颜色

- `render.rs`
  - 调用 `figlet-rs`
  - 将输入文本渲染为 ASCII 艺术字字符串

- `color.rs`
  - 将渲染结果按指定颜色输出到终端

## CLI 设计
第一阶段命令行接口：

- 位置参数：`text`
- 可选参数：`--color <name>`

示例：

```bash
cargo run -- hello
cargo run -- hello --color red
cargo run -- "hello world" --color cyan
```

## 颜色范围
第一阶段只支持固定颜色名称：

- `red`
- `green`
- `blue`
- `yellow`
- `cyan`
- `magenta`
- `white`

默认颜色：

- 未传 `--color` 时，使用 `white`

## 输出规则
- 输入有效文本后，程序输出对应的大号 ASCII 艺术字
- 整块输出使用同一种颜色
- 如果文本为空或全是空白，程序报错并退出
- 如果颜色值不合法，程序报错并退出

## 验收标准
满足以下条件即可认为第一阶段完成：

- 能从命令行读取文本参数
- 能生成大号 ASCII 艺术字
- 能按指定颜色输出
- 默认颜色行为正确
- 非法输入有明确错误提示
- 在当前终端环境下可正常运行

## 后续计划
第二阶段再继续扩展：

- 单色之外的左到右渐变
- 更丰富的颜色方案
- 动画效果
- 是否切换到自定义字模渲染器，再根据需求评估
